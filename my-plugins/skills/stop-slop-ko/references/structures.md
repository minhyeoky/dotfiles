# 피할 구조 (한국어)

## 이항 대조 (가짜 드라마)

부정으로 긴장을 만든 뒤 뒤집는 틀. 영어 "not X but Y"의 한국어판이다. Y를 바로 말한다.

| 패턴 | 문제 |
|------|------|
| "단순히 X가 아니라 Y다" | 예고된 반전 |
| "X가 아닌 Y이다" | 기계적 재구성 |
| "문제는 X가 아니다. Y다." | 공식화된 리프레임 |
| "X처럼 보이지만 사실은 Y다" | 셋업/반전 클리셰 |
| "중요한 건 X가 아니라 Y다" | 수사적 오도 |
| "X보다는 오히려 Y" | 같은 공식, 다른 말 |
| "X를 넘어 Y로" | 가짜 전환 서사 |
| "X일 뿐 아니라 Y이기도 하다" | 덧붙이기 헤지 |

**대신:** Y를 바로 말한다. "Y가 중요하다." 부정을 통째로 버린다.

## 부정 나열 (수사적 스트립쇼)

무엇이 *아닌지* 늘어놓고 마지막에 *무엇인지* 밝히는 틀.

| 패턴 | 문제 |
|------|------|
| "X도 아니고, Y도 아니다. Z다." | 부정을 통한 극적 빌드업 |
| "X가 아니었다. Y도 아니었다. Z였다." | 같은 구조, 과거형 |

**대신:** Z를 바로 말한다. 독자는 활주로가 필요 없다.

## 극적 단문 (꾸며낸 깊이)

짧은 단문·체언종결을 강조용으로 쌓으면 인위적인 비장함이 된다.

| 패턴 | 문제 |
|------|------|
| "그게 전부다. 그게 핵심이다." | 연출된 단순함 |
| "X. 그리고 Y. 그리고 Z." | 스타카토 드라마 |
| "이것이 모든 걸 바꾼다. 정말로." | 인위적 폭로 |
| 명사·체언으로 끊는 단문 연속 ("성장. 그리고 정체.") | 비장한 척 |

**대신:** 완결된 문장. 연출보다 내용을 믿는다.

## 종결어미 단조 (한국어 특유의 리듬 문제)

"~입니다 / ~합니다"만 반복하면 기계가 찍어낸 리듬이 된다. 한국어 AI 글의 가장 흔한 티.

| 패턴 | 고치기 |
|------|--------|
| "~입니다. ~입니다. ~합니다." 연속 | 평서·명사형·의문을 섞는다 |
| 모든 문장이 같은 길이 | 길고 짧음을 섞는다 |
| "~하고, ~하며, ~한데, ~하여..." 만연체 | 끊어 짧은 문장으로 나눈다 |
| "~것이다 / ~것입니다" 남발 | 동사로 바로 끝낸다 |

**대신:** 세 문장이 같은 종결어미면 하나를 바꾼다. 길이도 흔든다.

## 만연체 — 쉼표로 절 잇기 (일본어 장문 직역)

위 "종결어미 단조"의 만연체는 `~하고/~하며/~한데` 같은 **연결어미** 늘어짐을 다룬다. 여기서는 **쉼표(,)로 절을 끝없이 잇는** 별개 증상을 잡는다. 일본어 장문을 한 문장으로 옮기면 난다. 한국어 독자는 한 문장에 절 하나~둘이 편하다.

| 패턴 | 고치기 |
|------|--------|
| "A는 B이고, C는 D이며, E는 F인데, 그래서 G다" (쉼표로 절 3+ 연결) | 마침표로 2~3문장으로 쪼갠다 |
| 주어가 중간에 바뀌는데 한 문장으로 묶음 | 주어 바뀌는 지점에서 끊는다 |
| 쉼표 뒤마다 새 정보가 무한 증식 | 핵심 한 절만 본문장, 나머지는 다음 문장으로 |

**대신:** 한 문장에 절이 셋 이상 쌓이면 끊는다. 끊으면 종결어미 변주 기회도 생긴다(같은 격식 안에서). 단, 의도된 긴 호흡까지 기계적으로 자르면 글이 토막 난다 — 절 셋 이상이 *정보를 무한 추가*할 때만 끊는다.

## 무생물 주어 (행위자 숨기기)

무생물에 사람 동사를 붙인다. 데이터는 말하지 않고, 시장은 보상하지 않는다. 누군가 그렇게 만든다. AI는 행위자 이름을 피하려고 이 틀을 좋아한다.

| 패턴 | 문제 |
|------|------|
| "데이터가 말해준다 / 보여준다" | 데이터는 가만히 있다. 누군가 읽고 해석한다. |
| "시장이 보상한다 / 응답한다" | 시장이 아니라 사는 사람이 돈을 낸다. |
| "이 기술이 ~을 가능하게 한다" | 기술이 아니라 사람이 그걸로 한다. |
| "결정이 내려진다 / 도출된다" | 결정은 저절로 안 난다. 누군가 결정한다. |
| "문화가 바뀐다" | 문화가 아니라 사람이 행동을 바꾼다. |
| "이 결과는 ~을 시사한다" | 결과가 아니라 당신이 그렇게 읽는다. |

**대신:** 사람을 댄다. "팀이 그 주에 고쳤다"가 "문제가 해결되었다"보다 낫다. 댈 사람이 없으면 "당신"으로 독자를 자리에 앉힌다.

## 유정/무정 동사 불일치 (무정물에 사람 전용 동사)

'무생물 주어'가 주어 자리를 본다면, 이건 **동사 선택**을 본다. 한국어는 주어의 유정성(생물/무생물)에 따라 쓸 수 있는 동사가 갈린다. 무정물에 사람·동물 전용 동사를 붙이면 번역기 티가 난다. 영어 die/born/survive/suffer를 사물에도 직역한 결과다.

| 패턴(무정물 + 사람 동사) | 고치기(무정물 동사) |
|------|------|
| "그 나무는 사망했다 / 죽음을 맞이했다" | "그 나무는 말라 죽었다 / 고사했다" |
| "이 프로젝트가 탄생했다 / 태어났다" | "이 프로젝트가 시작됐다 / 생겼다 / 만들어졌다" |
| "그 기능은 사망했다 / 숨졌다" | "그 기능은 빠졌다 / 사라졌다 / 없어졌다" |
| "구버전 API가 살아남았다 / 생존했다" | "구버전 API가 남아 있다 / 유지됐다" |
| "이 서비스가 고통받고 있다 / 신음한다" | "이 서비스가 느리다 / 자주 죽는다 / 문제가 많다" |
| "데이터가 살아 숨 쉰다" | "데이터가 계속 갱신된다" (대개 삭제) |

**판별:** 모어 화자가 일상에서 그 결합을 쓰나? 굳은 비유 관용("회사가 살아남았다", "프로젝트가 죽었다", "전통이 살아 있다")은 자연스러운 한국어다 — 건드리지 않는다. 문제는 **굳지 않은 직역**("나무가 사망했다", "기능이 숨졌다")뿐이다.

**대신:** 무정물엔 무정물 동사("생기다 / 만들어지다 / 빠지다 / 사라지다 / 남다 / 망가지다 / 멈추다"). 사람·동물엔 사람·동물 동사("태어나다 / 죽다 / 사망하다 / 살아남다").

## 지나친 의인화 (마음 없는 것에 마음을 준다)

'무생물 주어'는 주어 자리(행위자 은폐)를, '유정/무정 동사'는 생물학 동사 직역(die/born)을 봤다. 세 번째 결은 **마음 부여**다 — 인지·의지·감정 같은 *마음의 작용*을 마음 없는 것(AI·모델·시스템·코드·데이터·알고리즘)에 붙인다. 주어도 명시돼 있고 생물학 동사도 아니라 앞 두 항목에 안 걸리지만, AI·기술 글에서 가장 흔한 티다.

| 패턴(마음 부여) | 고치기(기계 동작) |
|------|------|
| "모델이 이해한다 / 안다 / 생각한다 / 추론한다 / 깨닫는다" | 실제 연산으로: "~로 분류한다 / 출력한다 / ~ 패턴에 반응한다" |
| "AI가 원한다 / 의도한다 / 노력한다 / 똑똑하게 판단한다" | 의지·지능 부여 삭제: "이 함수는 ~할 때 ~한다" |
| "코드가 알고 있다 / 기억한다 / 신경 쓴다" | "이 값이 ~에 저장돼 있다 / ~을 참조한다" |
| "데이터가 스스로 학습한다 / 진화한다" | 사람을 행위자로: "이 모델을 ~ 데이터로 다시 학습시킨다" |
| "시스템이 깨닫는다 / 느낀다 / 고민한다" | "조건이 ~이면 ~로 분기한다" |
| 감정 의인화 "외로운 서버 / 지친 DB / 행복한 코드" | 시적 의인화. 기술 글에선 삭제, 비유 의도가 분명하면 1회만 |

**판별:** 그 동사가 *마음의 작용*(이해·욕망·감정·자각)인가, *기계의 동작*(분류·반환·갱신)인가? 마음의 작용을 마음 없는 것에 붙였으면 기계 동작으로 바꾼다.

**경계(도메인 표준어는 의인화가 아니다):** 분야에서 굳은 표준 용어와 관용은 건드리지 않는다.
- ML 표준: "모델이 학습한다 / 수렴한다 / 예측한다 / 추론(inference)한다" — 자연스럽다
- 기술 표준: "함수가 ~을 반환한다 / 서버가 응답한다 / 프로세스가 죽었다 / 프로그램이 실행된다 / API가 ~을 제공한다"
- 표적은 **굳지 않은 마음 부여**뿐. "모델이 학습한다"(O) vs "모델이 이해하고 싶어 한다"(X). 같은 "학습"도 표준 술어면 두고, "스스로 깨우친다"는 의지 부여면 뺀다.

## 잉여 주어·대명사 (영어가 강요한 것)

영어는 모든 절에 주어가 있어야 하고("It is...", "I love you", "They say"), 같은 대상을 가리킬 때 대명사(it/he/she/they)를 반복한다. 한국어는 맥락으로 주어를 생략하고, 대명사 대신 무주어나 명사 반복을 쓴다. 영어 직역은 "그것은 / 나는 / 그는 / 우리는"을 문장마다 깐다.

| 패턴(잉여 주어·대명사) | 고치기 |
|------|------|
| "그는 말했다. 그는 떠났다. 그는..." (he 반복) | 두 번째부터 주어 생략: "말하고 떠났다" |
| "이 라이브러리는 가볍고, 그것은 빠르며, 그것은 문서도 좋다" | 대명사 사슬을 끊는다: "가볍고 빠르고 문서도 좋다" |
| "그들은 그것이 작동한다고 말한다" (they say it) | "작동한다고들 한다 / 작동한다고 알려져 있다" |
| 한 문단에서 같은 대상에 "그 / 그것 / 이것"을 매 문장 반복 | 첫 등장 후 생략하거나 명사로 한 번만 |

(단순 "그것은 ~이다"(it is), "당신의 ~"(your/you)는 phrases.md 번역투 표에 이미 있다. 여기서는 **대명사 사슬 반복**과 일반 주어 "나는/우리는/그는" 생략에 초점.)

**경계(과잉 생략 금지):** 주어를 빼면 누구/무엇인지 헷갈리는 곳은 남긴다. 화제가 바뀌는 첫 문장, 대조하는 두 주체("나는 찬성, 그는 반대"), 강조 초점("그건 네 문제야")은 주어가 의미를 진다. 빼는 건 **맥락으로 100% 복원되는** 주어·대명사뿐이다. "그것/이것"이 가리키는 게 멀거나 여럿이면 빼지 말고 명사로 되살린다(생략이 아니라 명시).

**대신:** 직전 문장에서 자명한 주어는 떨군다. 대명사 사슬은 끊고 필요한 한 번만 명사로 받는다.

## 피동·이중피동 (행위자를 지우는 말)

문장마다 행위자가 있어야 한다. 피동은 행위자를 숨기고 힘을 뺀다. 이중피동은 무조건 틀린다.

| 패턴 | 고치기 |
|------|--------|
| "보여진다 / 쓰여진다 / 잊혀진다 / 닫혀진다" (이중피동) | "보인다 / 쓰인다 / 잊힌다 / 닫힌다" |
| "생각되어진다 / 만들어진다 / 되어진다" | "생각된다 / 만들어진다"는 OK지만 능동이 낫다 |
| "~에 의해 만들어졌다" | "~가 만들었다" |
| "~라고 여겨진다 / 평가된다" | 누가 그렇게 보는지 댄다 |
| "결정이 내려졌다" | "~가 결정했다" |

**대신:** 행위자를 찾아 문장 앞에 세운다.

## 수사적 셋업 (통찰을 전달하지 않고 예고만)

| 패턴 | 문제 |
|------|------|
| "만약 ~라면 어떨까?" | 소크라테스 흉내 |
| "무슨 말이냐면" | 불필요한 예고 |
| "한번 생각해보자 / 생각해보라" | 가르치려는 명령 |
| "그래도 괜찮다 / 그것도 나쁘지 않다" | 불필요한 허락 |

**대신:** 요점을 말한다. 결론은 독자가 내리게 둔다.

## 균형 강박 (모든 것에 양면을 다는 버릇)

AI는 단정을 피하려고 반대 측면을 기계적으로 붙인다. 글이 흐물거린다.

| 패턴 | 문제 |
|------|------|
| "물론 ~한 측면도 있다" (자동반사) | 가짜 균형 |
| "다만 / 하지만 ~할 수도 있다" (남발) | 단정 회피 |
| "경우에 따라 다를 수 있다" | 책임 회피 |
| "~인 것으로 보인다 / ~인 듯하다" (남발) | 흐릿한 마무리 |

**대신:** 입장이 있으면 단정한다. 진짜 반론만, 필요할 때만 단다.

## 접속사·연결어 남발

| 패턴 | 고치기 |
|------|--------|
| 문장마다 "또한 / 그리고 / 하지만 / 따라서 / 한편" | 대부분 지운다. 한국어는 조사·어미로 연결된다. |
| "뿐만 아니라 / 더 나아가" 반복 | 한 번이면 족하다 |

## 리듬·나열

| 패턴 | 고치기 |
|------|--------|
| 무조건 세 가지 나열 ("첫째, 둘째, 셋째") | 두 개나 한 개로 충분한지 본다 |
| 모든 것을 bullet으로 | 산문이 맞으면 산문으로 |
| 질문하고 바로 답하기 | 질문이 숨 쉬게 두거나 질문을 뺀다 |
| 문단마다 한 줄 펀치라인으로 끝 | 맺음을 다양하게 |

## 단어 패턴

| 패턴 | 문제 |
|------|------|
| 막연한 극단어 ("모든 / 항상 / 절대 / 누구나 / 전부") | 가짜 권위. 구체로 바꾼다. |
| 부사 전반 ("정말 / 매우 / 사실 / 결국 / 단순히") | 공허한 강조. phrases.md 참고. |
| 한자 추상명사 남발 ("~성 / ~화 / ~적") | 동사·형용사로 푼다 |

## 강조·서식 남발 (마케팅 카피 티)

문장 내용이 아니라 서식으로 무게를 주려는 버릇. 랜딩페이지 카피를 한국어에 그대로 부으면 난다.

| 패턴 | 문제 |
|------|------|
| 볼드를 거의 매 문장에 하나씩 | 강조가 전부면 강조가 없다 |
| 양쪽 em-dash 삽입 "— 삽입구 —" | 영어 번역투. 한국어는 쉼표·괄호 |
| 가운뎃점 나열 "A·B·C·D"가 문서 박자로 반복 | 3·4항 묶음의 강박. 한 번이면 족 |
| 명사형 의문절을 볼드로 "어떻게 ~하는가를 다룬다" | 심오한 척. 구체 통증을 말한다 |
| "누구를 위한 X" + 타깃 불릿 3줄 | 마케팅 덱 골격. 산문 한 문단으로 |
| "~를 다룬다 / ~가 무대다" 거리감 화법 | 해설하지 말고 동료에게 말하듯 |

**대신:** 무게는 단어 선택과 구체성으로 준다. 서식은 거들 뿐, 내용을 대신하지 않는다.

## 접속부사 뒤 쉼표 (영어 구두점 직역)

문두 접속부사 바로 뒤에 쉼표를 찍는 버릇. 영어는 "Therefore, / However, / Also,"처럼 문두 접속부사 뒤에 쉼표를 의무로 찍지만, 한국어 접속부사는 그 자체로 한 호흡이라 쉼표가 필요 없다. 글 전체에서 반복되면 영어 규칙을 기계가 옮겨 적은 티가 난다.

| 패턴 | 고치기 |
|------|--------|
| "따라서, ~" / "그러므로, ~" | "따라서 ~" / "그러므로 ~" |
| "하지만, ~" / "그러나, ~" | "하지만 ~" / "그러나 ~" |
| "또한, ~" / "게다가, ~" | "또한 ~" / "게다가 ~" |
| "그리고, ~" / "그래서, ~" | "그리고 ~" / "그래서 ~" |
| "한편, ~" / "반면, ~" | "한편 ~" / "반면 ~" |
| "즉, ~" / "결국, ~" | "즉 ~" / "결국 ~" (부사 자체 남발은 별도 점검) |

**대상은 접속부사 *직후* 쉼표뿐이다.** 병렬("빠르고, 가볍고, 안정적이다")·삽입구("이 방식은, 솔직히, 느리다")·종속절 뒤 호흡 쉼표는 한국어에서 제 일을 하므로 그대로 둔다. 모든 쉼표를 적대하지 않는다.

**대신:** 접속부사는 두고 쉼표만 뗀다. 접속부사 자체가 매 문장 반복되면 그건 "접속사·연결어 남발" 항목에서 따로 줄인다(이 규칙은 구두점만 고친다).

## "~의" 연쇄 (단어 패턴 보강)

"한자 추상명사(~성/~화/~적)"에 더해, 조사 **"~의"(の) 연쇄**도 일본어투 신호다. 상세·교정은 phrases.md "~의 남용" 참고. 여기서는 신호만: *한 구에 "~의"가 둘 이상이면 일본어투를 의심한다.*
