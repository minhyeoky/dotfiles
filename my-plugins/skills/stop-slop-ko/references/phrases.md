# 걷어낼 표현 (한국어)

## 군더더기 서두 (예고만 하는 말)

본론을 예고하는 말. 잘라내고 바로 본론을 친다.

- "결론부터 말씀드리면"
- "먼저 ~부터 살펴보겠습니다 / 알아보겠습니다"
- "본 글에서는 / 이 글에서는"
- "지금부터 ~"
- "한 가지 짚고 넘어갈 점은"
- "여기서 중요한 것은"
- "사실을 말하자면 / 솔직히 말해서"
- "다름 아니라"
- "다들 아시다시피 / 알다시피"
- "~에 대해 이야기해보자 / 말씀드리자면"

"~에 대해 알아보겠습니다" 류는 전부 본론 앞의 헛기침이다. 지우고 본론을 말한다.

## 챗봇 추임새 (AI 특유의 공손·아부)

대화형 AI의 가장 뚜렷한 티. 정보를 더하지 않는다.

- "물론입니다! / 그럼요!"
- "좋은 질문입니다 / 훌륭한 지적입니다"
- "~하는 것은 훌륭한/좋은 선택입니다"
- "함께 살펴볼까요? / 알아볼까요?"
- "~해보는 건 어떨까요?" (정보 전달 글에서)
- "도움이 되었기를 바랍니다 / 도움이 되셨으면 좋겠습니다"
- "더 궁금한 점이 있으면 언제든 물어보세요"
- "걱정 마세요 / 안심하세요"
- "~라는 점, 꼭 기억하세요"

## 강조 크러치 (힘주는 척하는 말)

의미를 더하지 않는다. 삭제한다.

- "정말 중요합니다 / 매우 중요한"
- "꼭 / 반드시 / 무조건" (남발 시)
- "그 무엇보다 / 다른 무엇보다"
- "두말할 나위 없이 / 말할 것도 없이"
- "명심해야 할 것은"
- "강조하고 싶은 것은 / 다시 한번 강조하지만"

## 번역투 표현 (영어 직역에서 온 어법)

한국어에 원래 없던 영어식 어법. 한국어 어순·조사로 다시 쓴다.

| 번역투 | 한국어로 |
|--------|----------|
| "~을 가지고 있다 / 가지다" (have) | "~이 있다", "~을 한다" |
| "~에 의해 (만들어졌다)" (by) | 행위자를 주어로: "~가 만들었다" |
| "~을 통해 / ~를 통하여" (through) | "~로", "~으로써", 또는 동사로 풀기 |
| "~에 대한 / ~에 대해" (about, 남발) | 조사로: "~의", "~을", "~를" |
| "~에 있어서 / ~에 있어" (in) | "~에서", "~에는", 삭제 |
| "~로부터" (from, 남발) | "~에게서", "~에서" |
| "가장 ~한 것 중 하나" (one of the most) | "손꼽히는 ~", "~한 편이다" |
| "~할 필요가 있다" (need to, 남발) | "~해야 한다" |
| "당신의 ~ / 당신은" (your/you, 남발) | 생략하거나 맥락으로 |
| "그것은 ~이다 / 이것은 ~이다" (it is) | 주어를 명사로 바로, 또는 생략 |
| "~라는 사실 (that절 직역)" | "~라는 점", 또는 풀어쓰기 |
| "~되어진 / ~라고 불리는" | "~된", "~라는", "~인" |
| "~을 제공한다 (provide)" | "~을 준다", "~한다" |
| "~을 의미한다 (means, 남발)" | "~라는 뜻이다", 또는 바로 결론 |

## 부사·군더더기 (없어도 되는 수식)

빼도 뜻이 안 변하면 뺀다.

- "정말 / 진짜 / 너무 / 매우 / 굉장히 / 엄청"
- "사실 / 사실은 / 실제로"
- "결국 / 결국에는"
- "단순히 / 그저 / 단지"
- "기본적으로 / 근본적으로 / 본질적으로"
- "어느 정도 / 다소 / 약간 / 나름" (모호한 완충)
- "당연히 / 물론" (문장 머리 남발)
- "특히 / 무엇보다" (남발)
- "~인 셈이다 / ~라고 할 수 있다" (단정 회피)

## 명사화·추상화 (동사로 풀어야 할 것)

영어식 명사 중심 표현. 한국어는 동사로 말한다.

- "~의 중요성을 강조한다" → "~가 중요하다"
- "~의 필요성이 있다 / 제기된다" → "~해야 한다"
- "~의 가능성을 열어준다" → "~할 수 있게 한다"
- "~에 대한 이해를 높인다" → "~을 더 잘 이해하게 한다"
- "~화(化)를 진행한다" → "~한다" (자동화를 진행 → 자동화한다)
- "~성(性)" 추상명사 남발 → 형용사·동사로

## 메타 발화 (글이 자기 구조를 설명하는 말)

글은 움직여야지, 자기 차례표를 읽어선 안 된다. 삭제한다.

- "지금부터 ~을 설명하겠습니다"
- "이어서 / 다음으로 살펴볼 것은"
- "앞서 언급했듯이 / 위에서 살펴본 것처럼"
- "본격적으로 들어가기 전에"
- "정리하자면 / 요약하자면" (마무리 자동반사)
- "이번 글의 핵심은"

## 공허한 마무리

스스로 맺지 못하고 마무리를 선언하는 말. 마지막 문장이 내용으로 끝나게 한다.

- "결론적으로 / 종합하면 / 결론은"
- "이처럼 / 이렇듯 / 이와 같이"
- "다시 말해 / 즉 (남발)"
- "지금까지 ~에 대해 알아보았습니다"
- "앞으로가 더욱 기대된다 / 귀추가 주목된다"

## 모호한 선언 (구체를 안 대고 중요하다고만 하는 말)

무엇이 어떻게 중요한지 안 대고 "크다/많다"고만 한다. 잘라내거나 구체로 바꾼다.

- "그 의미는 크다 / 의미가 깊다"
- "시사하는 바가 많다 / 시사점이 크다"
- "파장이 적지 않다"
- "주목할 만하다 / 눈여겨볼 대목이다"
- "결코 가볍지 않은 문제다"

구체적인 그것을 못 대면 문장을 지운다.

## 일본어투 관용구 직역 (일본어 어법을 그대로 옮긴 말)

번역투(영어)와 별개로, 한국어 글에 스며든 일본어 직역체. 한국어는 더 짧고 직설로 단정한다.

| 일본어투 | 한국어로 |
|----------|----------|
| "~임에 틀림없다 / ~에 틀림없다" (に違いない) | "~일 것이다", "분명 ~다", "~가 확실하다" |
| "~하지 않으면 안 된다 / ~지 않으면 안 된다" (なければならない) | "~해야 한다" |
| "~에 다름 아니다 / ~에 다름없다" (にほかならない) | "바로 ~다", "~일 뿐이다" |
| "~라고 하지 않을 수 없다" (と言わざるを得ない) | "~다", "~라고 봐야 한다" |
| "~라고 말할 수 있을 것이다" (と言えるだろう) | "~다" (단정 회피 군더더기) |
| "~에 임하다" (臨む, 남발) | "~에 나서다", "~을 하다" |

**주의:** "~할 필요가 있다"(need to)는 영어투로 위 번역투 표에 이미 있다. 둘 다 "~해야 한다"로 가지만 출처가 다르다 — 헷갈리지 말 것. "~라고 말할 수 있을 것이다"도 "~라고 할 수 있다"(번역투/단정 회피)의 일본어 변종이다.

### "~의"(の) 남용 — 조사 "~의" 연쇄

일본어 の는 명사를 무한히 잇지만, 한국어 "~의"는 한 구에 한 번이면 충분하고 자주 생략한다. 명사 두 개가 자연스레 붙으면(복합어·수식) "~의"를 지운다.

- "최고의 성능의 제품" → "성능이 가장 좋은 제품"
- "AI의 발전의 속도" → "AI 발전 속도"
- "우리의 회사의 목표" → "우리 회사 목표"
- "최근의 시장의 동향" → "최근 시장 동향"
- "그녀의 미소의 의미" → "그녀의 미소가 뜻하는 것"

**판별:** 한 구(절 안)에 "~의"가 둘 이상이면 의심한다. 소유·소속이 꼭 필요한 한 곳만 남기고 나머지는 지우거나 동사절로 푼다. "~의" 자체가 틀린 게 아니다 — *연쇄*가 일본어투다. (잡는 것은 빈도다 → SKILL.md 빈도 임계값 참고.)

## 자기칭찬·자랑 (제 결과물을 스스로 추켜세우는 말)

앞의 세 뿌리가 *어법*이라면, 이건 *내용*이다. AI는 제가 만든 것에 칭찬 형용사를 스스로 붙인다. 품질은 글쓴이가 선언하는 게 아니라 독자가 사실을 보고 판단한다. 자평 형용사는 지우고, 근거가 있으면 형용사 대신 그 근거(수치·동작·결과)를 댄다. 강한 신호 — 1회라도 뺀다.

- 자평 형용사: "우아한 / 정교한 / 강력한 / 견고한 / 탄탄한 / 세련된 / 깔끔한(자평) / 완벽한 / 최적의 / 혁신적인 / 획기적인 / 차세대 / 압도적인"
- 영어 AI 형용사 직역: "seamless한 / robust한 / powerful한 / elegant한 / scalable한"
- 성취 과장: "완벽하게 해결했습니다 / 획기적으로 개선했습니다 / 한 차원 끌어올렸습니다 / 대폭 향상시켰습니다 / 근본적으로 바꿉니다"
- 자기 노동 광고: "공들여 / 심혈을 기울여 / 꼼꼼하게 다듬은 / 정성껏"
- 자기 위치 과시: "업계 최고 수준의 / 타의 추종을 불허하는 / 누구나 인정하는"

**판별:** "누가, 어떻게 보여줬나?"를 물었을 때 답이 글 안에 없으면 자기칭찬이다 — 지운다. 답이 있으면(벤치마크·동작·결과) 형용사를 버리고 그 답만 남긴다. "강력한 캐싱"이 아니라 "반복 요청을 90% 줄이는 캐싱". (외부 대상을 *근거 있게* 칭찬하는 건 사실 진술이라 건드리지 않는다 → SKILL.md 가드레일.) 모호한 선언("그 의미는 크다")과는 다르다 — 그건 중요성 부풀리기, 이건 제 결과물 품질 자랑이다.

## 메타정보 누수 (제작 맥락이 완성 글에 새는 것)

대화·메모리·지시처럼 *글을 만든 과정*에만 있던 맥락을 완성 글에 흘린다. 최종 글의 독자는 그 과정을 본 적이 없다 — 글은 그 자체로 자족해야 한다. 강한 신호 — **완성 글**에서는 1회라도 뺀다. (사용자에게 직접 답하는 *대화*에서 앞 맥락을 가리키는 건 정상이니 건드리지 않는다.)

- 대화 참조: "앞서 말씀하신 대로 / 위에서 논의한 것처럼 / 아까 ~라고 하셨듯이"
- 지시 되받기: "요청하신 대로 / 말씀하신 ~를 반영하여 / ~를 개선해 달라고 하셨는데"
- 제작 과정 서술: "이 문서를 작성하면서 / 제가 ~를 만들었습니다 / 이번에 새로 정리한 / 다음과 같이 구성했습니다"
- 개인·맥락 누수: 독자의 목적과 무관한데 시스템 컨텍스트(메모리·프로필)에서 흘러든 정보 — 일반 가이드에 갑자기 "당신의 tmux+nvim 환경에서는", 무관한 글에 "전 직장에서 일하셨으니"
- 청중 혼동: 일반 독자용 글인데 대화 상대를 가리킴 — "당신이 물어보신 것처럼"

**판별:** 이 글을 처음 보는 사람이 읽었을 때 "뭘 가리키는 거지?" 하고 막히면 누수다 — 뺀다. 단, 독자를 "당신"으로 호명하는 수사(규칙 3의 독자 자리 앉히기)는 누수가 아니다 — 그건 제작 과정이 아니라 글의 독자를 가리킨다. 글쓴이가 의도해 밝힌 출처("이 글은 사내 위키를 정리한 것")도 글쓴이의 선택이지 누수가 아니다.
